# Security Policy

## Supported Versions

The following versions currently receive security updates.

| Version | Supported |
| ------- | --------- |
| 1.x.x   | Yes       |
| 0.x.x   | No        |

---

# Reporting a Vulnerability

The security of INFERA is taken seriously.

If you discover a security vulnerability, please do not create a public GitHub issue.

Instead, report the issue privately.
  to belmiredub4@gmail.com
  
---

## What To Include

Please provide:

* Description of the vulnerability
* Steps to reproduce
* Potential impact
* Suggested fix (if available)
* Screenshots or logs (if applicable)

---

## Response Timeline

After receiving a report:

### Within 72 Hours

* Initial acknowledgment

### Within 7 Days

* Investigation begins

### Within 14 Days

* Status update provided

### As Soon As Possible

* Security patch released

---

## Scope

Examples of security issues include:

* Authentication bypass
* Authorization flaws
* Session vulnerabilities
* SQL Injection
* Cross-Site Scripting (XSS)
* CSRF vulnerabilities
* Sensitive data exposure
* Privilege escalation

---

## Out of Scope

The following are generally considered out of scope:

* UI bugs
* Typographical errors
* Feature requests
* Vulnerabilities in outdated unsupported versions
* Third-party dependency issues without a practical exploit path

---

## Responsible Disclosure

Please allow reasonable time for investigation and remediation before publicly disclosing any vulnerabilities.

Responsible disclosure helps protect users and maintain the security of the platform.

Thank you for helping improve the security of INFERA.
